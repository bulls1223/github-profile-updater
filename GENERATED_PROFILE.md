## 🚀 Getting Started

로컬 개발 환경에서 프로젝트를 실행하기 위한 가이드입니다.

### Prerequisites
- **JDK 1.8** 이상
- **Oracle Database** (로컬 또는 개발 DB 연결 필요)
- **Gradle** (Wrapper 내장)

### Installation

1. 리파지토리를 클론합니다.
    ```bash
    git clone <repository-url>
    cd vlu-platform
    ```

2. 프로젝트 의존성을 설치하고 빌드합니다.
    ```bash
    gradlew clean build -x test
    
    ./gradlew clean build -x test
    ```

### Run

메인 웹 어플리케이션(`vlu-web`)을 실행합니다.

```bash
gradlew :vlu-web:bootRun

./gradlew :vlu-web:bootRun
```

실행 후 브라우저에서 `http://localhost:8080` 으로 접속할 수 있습니다.

## 💡 Core Logic Description

**TradingService** (`com.valuelinku.web.trading.service`)

이 프로젝트의 핵심인 부킹/거래(Trading) 프로세스를 담당하는 서비스입니다. 주요 로직은 다음과 같습니다:

- **SPO 생성 및 검증 (`validateSpo`, `createSpo`)**: 
  - 사용자의 화물 정보(DG, RF, General)와 선복량(Space)을 체크하여 부킹 가능 여부를 검증합니다.
  - `BkgReqDt`(요청일), `CrrCd`(운송사 코드) 등을 기반으로 고유한 `ReqNo`(요청번호)를 생성합니다.
  - 위험물(DG), 냉동/냉장(RF) 화물에 대한 별도 테이블 저장 로직을 수행합니다.
  
- **물류 서비스 연동**: 
  - 부킹 생성 시 트럭(Truck), 통관(Clearance), 보험(Insurance) 서비스 신청 여부를 확인하고, 관련 데이터를 함께 처리합니다.
  - 파일 업로드(`ComFileService`)와 연계하여 선적 서류 및 증빙 파일을 관리합니다.

- **상태 관리**: 
  - 부킹의 생애 주기(임시저장 -> 요청 -> 확정 -> 취소)에 따라 상태 코드(`BkgReqStsCd`)를 관리하고 이력을 남깁니다.

## 🧠 Core Logic: Synchronization Flow

이 프로젝트의 핵심은 **`Controller`가 Trigger 역할을 하고, `Service`가 동기화 로직을 수행**하는 구조입니다.

1.  **Triggering (`Controller`)**:
    *   `com.valuelinku.controller` 패키지 내의 클래스들은 `@Scheduled` 어노테이션을 사용하여 Cron 표현식에 따라 작업을 예약합니다.
    *   예: `AirOnSyncController`는 매주 월요일 특정 시간에 항공 운임 복사 로직을 호출합니다.

2.  **Processing (`Service`)**:
    *   `com.valuelinku.service` 내의 서비스 클래스(예: `AcademySyncService`)는 실제 비즈니스 로직을 담당합니다.
    *   **Data Fetching**: Source DB(예: Main)에서 JPA Repository 등을 통해 최신 데이터를 조회합니다.
    *   **Data Conversion**: `VluCommonUtil`을 사용하여 Source 객체를 Target 객체 포맷으로 변환합니다.
    *   **Synchronization**: Target DB(예: Academy)의 기존 데이터를 정리(Delete)하거나 중복을 체크한 후, 변환된 데이터를 저장(Save)합니다.

```java
// Logic Example: AcademySyncService.java
@Transactional
public void syncUser() {
    // 1. 최근 변경된 데이터 조회 (Source DB)
    List<ComUser> users = acdmUserRepo.findAllByRegDtAfter(date);
    
    // 2. 데이터 존재 여부 확인 및 동기화 (Target DB)
    for (ComUser user : users) {
        if (!mainUserRepo.existsById(user.getUsrNo())) {
            mainUserRepo.save(VluCommonUtil.convertComUser(user));
        }
    }
}
```

## 🧠 Core Logic: Lashing System

이 프로젝트의 핵심인 `LashingPage`는 **SVG 좌표계 연산**을 통해 화물 고정 작업을 디지털 트윈(Digital Twin)화 합니다.

*   **Coordinate System (좌표계)**: 
    화물(Inner)과 컨테이너(Outer)의 3차원 공간을 2D 평면(Front/Back/Left/Right)으로 펼쳐서 SVG 상에 매핑합니다. `LashingPage.tsx`에서 각 Lashing Point(고리)의 좌표를 동적으로 계산하여 렌더링합니다.
    
*   **Validation Logic**: 
    `handleMouseUp` 이벤트를 통해 사용자가 유효하지 않은 연결(예: 같은 면끼리의 연결 등 물리적으로 불가능한 래싱)을 시도할 경우 이를 자동으로 차단하거나 보정합니다.

*   **Data Persistence**: 
    생성된 래싱 데이터는 JSON 형태로 직렬화되어 MySQL DB의 `vos_oog_lashing` 테이블에 `LASH_DATA` 컬럼으로 저장되며, 조회 시 다시 파싱되어 시각화됩니다.

## 💡 Core Logic Highlight

### Dynamic MyBatis Mapper Reloading (`RefreshableSqlSessionFactoryBean`)

이 프로젝트의 주목할 만한 기술적 특징은 **MyBatis 매퍼의 동적 리로딩**입니다.

- **문제점**: 보통 MyBatis XML 쿼리를 수정하면 JVM을 재시작해야 반영되어 개발 속도가 저하됩니다.
- **해결책**: `RefreshableSqlSessionFactoryBean`은 파일 시스템의 `lastModified` 타임스탬프를 주기적으로 감지합니다.
- **작동 원리**:
  1. `TimerTask`가 설정된 주기(기본 1000ms)마다 XML 리소스의 변경 여부를 체크합니다.
  2. 변경이 감지되면 `w.lock()`(Write Lock)을 걸고 `super.afterPropertiesSet()`을 호출하여 `SqlSessionFactory`를 리프레시합니다.
  3. 이를 통해 서버 중단 없이 쿼리 튜닝 및 수정이 가능합니다.

## 🧠 Core Logic

이 프로젝트의 핵심 로직은 `src/lib/converter` 디렉토리에 집중되어 있으며, 다음과 같은 파이프라인으로 동작합니다.

**`src/lib/converter/index.ts`**
변환 프로세스의 메인 컨트롤러로, 다음 단계들을 순차적으로 실행하여 최종 React 코드를 조립합니다.

1.  **Parsing & Model Extraction (`xmlParser`)**:
    -   초기 XML 문자열을 DOM으로 파싱합니다.
    -   WebSquare 고유의 `<w2:dataCollection>`을 찾아 DataMap, DataList 정의를 추출하여 데이터 모델을 구축합니다.
2.  **State Generation**:
    -   추출된 데이터 모델을 바탕으로 `useState` 훅 코드를 생성합니다.
3.  **Script Conversion (`scriptConverter`)**:
    -   XML 내부의 `<script>` 태그 내용을 분석하여 WebSquare API 호출을 이에 상응하는 React 로직(이벤트 핸들러 등)으로 변환합니다.
4.  **JSX Generation (`jsxConverter`)**:
    -   XML `<body>` 내부의 UI 요소들을 순회하며, 정의된 매핑 테이블(`constants/mappings`)을 참조해 적절한 React 컴포넌트 및 HTML 태그로 변환합니다.
5.  **Assembly**:
    -   위 과정에서 생성된 Import 구문, 타입 정의(Interface), State 선언, 이벤트 핸들러, JSX 구조를 하나의 완전한 기능수행 컴포넌트 문자열로 결합합니다.
